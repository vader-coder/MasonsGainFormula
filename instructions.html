Instructions for Mason's Gain Formula Website

The website can receive input through the table or as a .txt file. Use the buttons to add and remove rows on the table.

Pressing the 'Submit' button draws the relevant charts for the formula.

Each row in the input represents an edge. The website reads the list of edges from top to bottom.

The further to the top of the table or .txt file the first edge that contains a given node is, the further to the left in that node is graphed in the diagrams.

By default, the nodes are graphed along a horizontal axis but you can drag them to other locations; sometimes this renders the diagram more comprehensible.

Each edge's 'from' and 'to' nodes must be integers. The gain can consist of letters and numbers. For a subscript use an underscore (Gs as G_s).

Here are some examples of input files that will not work because they break one or more of the aforementioned rules:

0 1 a
3 2 b
2 3 c
2 4 d
0 1 a
1 2 b
2 5 c
5 6 d
6 3 e
3 4 f
6 5 g
3 2 h

0 must always exist as the source node, and there must always be at least one forward path from it to the sink node.

Each node must belong to at least one forward path connecting the source node to the sink node.

The sink node must have the largest integer label.

An edge is considered to point forwards if it points from a node with a lower integer to one with a higher integer.

An edge points backwards if it points from a node with a higher integer to one with a lower integer.

It is recommended (though not required) that you enter the edges in one path from the source to sink node with each edge's from = it's 'to'-1, if such a path exists, before entering the loops or forward paths between nodes with nonadjacent numbers to minimize the likelihood of having to drag the nodes around to make the diagrams more easily comprehensible.

Press 'Get Sample' for an example of a table and visit the GitHub repository for examples of .txt files.
